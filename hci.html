Carl-Mart

In the class Human Computer Interaction, I worked with three students, Wesley Yang, Khizar Qureshi, and Sho Tanaka in order to create an application that would solve a problem here at Carleton College. We decided to focus on waste reduction as well as providing opportunities for all individuals to get items necessary for college students. To do so we decided to create an application in which students can buy, sell, or trade items. This provides opportunities for students to get lightly used items that would normally be thrown away, from other students at cheap, affordable pricing. This project will help individuals that are possibly struggling with trasnportation or financial burderns in order to get items that they need for college.

Link to Figma Design

Changes for prototype II

After the first prototype of our figma application, we used fictional inquiry interviews in order to have a better understanding of what users want from our website through a UX process. Through a six step analysis process that included Initial axial coding, affinity diagram, creating a codebook, axial coding with our codebook, consensus building, and requirements generation, we were able to diagnose exactly what users were looking for in our application as well as how to apply them to our prototype. After such inquiries we were able to then continue with out prototyping with three requirements of what we should add to our prototypes.

Our analysis process included creating an initiala axial code, which was encoding interview transcripts of relevant passages. Then Creating an affinity diagram that held each of those axial encodings which we then created a codebook out of. This code book served as a key for us to look at and make decisions on what we needed to change and add. We then did consensus building where we all agreed on the best code and passages. Finally moving to requirement generations which are explained next.

From these inquiries, we identified our final three requirements: a Buy Page, Shopping Cart Page, and Sell Page. The Buy Page serves as the main platform for browsing and purchasing items, featuring tools like a filter button for easier navigation. The Shopping Cart Page lets users store items they’re interested in purchasing, avoiding the need to buy each item separately. The Sell Page provides a space for users to upload pictures, descriptions, and seller information, with features like payment type options.
